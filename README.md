# DearThemeLab

An interactive desktop application for designing [Dear ImGui](https://github.com/ocornut/imgui) themes visually — no recompile required.

Adjust colors, spacing, rounding, and every other style variable with live controls, see results instantly in a themed preview panel, then export the finished theme as ready-to-paste C++ code.

---

## Features

- **Live preview** — all changes reflect immediately in the Preview panel, which is rendered with your theme while the Editor panel stays readable no matter what
- **Comprehensive widget showcase** — the preview covers every ImGui widget type (buttons, sliders, inputs, tables, trees, menus, modals, plots, and more) so broken colors or spacing are immediately obvious
- **Full style editor** — every `ImGuiStyle` variable is exposed and grouped into collapsing sections: Main, Borders, Rounding, Alignment, Rendering, and Colors
- **Color filter** — search the color list by name to jump straight to the entry you want
- **C++ export** — generates a clean, paste-ready `ApplyTheme()` function with configurable options:
  - Custom function name and optional namespace
  - Toggle section comments on/off
  - Emit only values that differ from the Dark baseline
  - Adjustable float precision (1–4 decimal places)
- **Copy to clipboard** or **save to file** via a native file dialog
- **Dockable layout** — Preview and Editor panels are dockable and resizable; layout persists via `imgui.ini`

## Building

### Requirements

- CMake 3.16+
- C++17 compiler (Clang, GCC, or MSVC)
- Git (for dependency fetching)
- On **macOS**: Xcode Command Line Tools
- On **Linux**: GTK3 development headers (`libgtk-3-dev`)

All other dependencies (**Dear ImGui**, **GLFW**, **nativefiledialog**) are fetched automatically by CMake at configure time — no manual setup needed.

### Steps

```bash
git clone https://github.com/ancalagorn/DearThemeLab.git
cd DearThemeLab
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
./build/src/DearThemeLab
```

Or open the project folder in CLion and hit Run.

## Usage

1. **Launch the app** — Preview (left) and Editor (right) open docked side by side.
2. **Edit** — expand any section in the Editor panel and adjust values. The Preview updates live.
3. **Evaluate** — scroll through the showcase to check how the theme looks on every widget type.
4. **Export** — open the **Export** section, configure options, then copy to clipboard or save as a `.cpp` file.
5. **Use it** — paste the generated function into your project and call it once after `ImGui::CreateContext()`.

### Generated code example

```cpp
// Generated by DearThemeLab
// Paste this into your project and call ApplyTheme() once after ImGui::CreateContext()

#include <imgui.h>

void ApplyTheme()
{
    ImGuiStyle& style = ImGui::GetStyle();

    // Main
    style.WindowPadding     = ImVec2(8.00f, 8.00f);
    style.FramePadding      = ImVec2(4.00f, 3.00f);
    // ...

    // Colors
    style.Colors[ImGuiCol_Text]        = ImVec4(1.00f, 1.00f, 1.00f, 1.00f);
    style.Colors[ImGuiCol_WindowBg]    = ImVec4(0.06f, 0.06f, 0.06f, 0.94f);
    // ...
}
```

## Project Structure

```
src/
├── main.cpp          — GLFW + ImGui init, main loop
├── app.h / app.cpp   — App state (two ImGuiStyle instances), docking setup
├── preview_panel     — Preview dock, style swap logic
├── showcase          — Curated all-widgets panel
├── editor_panel      — Editor dock with all style sections
└── exporter          — Pure function: ImGuiStyle + options → C++ string
```
